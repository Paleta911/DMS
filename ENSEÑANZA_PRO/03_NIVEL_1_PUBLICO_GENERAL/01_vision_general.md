# Vision General del Sistema para Publico General

## De que trata el proyecto

El proyecto DMS SIG es una solucion web para gestionar documentos de forma ordenada, segura y trazable. Su razon de existir no es "digitalizar por digitalizar", sino resolver problemas concretos que aparecen cuando la documentacion de una organizacion crece y deja de poder controlarse de manera informal.

## Cual era el problema de fondo

Antes de un sistema de este tipo, la gestion documental suele depender de una mezcla poco estable de:

- carpetas compartidas,
- archivos sueltos,
- nombres poco consistentes,
- copias duplicadas,
- documentos obsoletos,
- controles manuales,
- y personas que saben "mas o menos" donde esta cada cosa.

Eso genera consecuencias muy concretas:

- se pierde tiempo buscando;
- cuesta saber cual version es la valida;
- no hay trazabilidad clara;
- la aprobacion de cambios es difusa;
- y se dificulta sostener auditorias o controles de calidad.

## Que propone el sistema

El sistema propone centralizar la documentacion en una sola plataforma, pero no solo como repositorio. Lo que realmente aporta es control.

Ese control se expresa en varios niveles:

- control de acceso;
- control por permisos;
- control por area;
- control de versiones;
- control del flujo de revision y aprobacion;
- control de evidencias mediante auditoria;
- y control de recuperacion de informacion mediante busqueda avanzada.

## Que puede hacer un usuario comun

Dependiendo de sus permisos, un usuario puede:

- entrar al sistema;
- ver su perfil;
- consultar documentos;
- subir documentos;
- subir nuevas versiones;
- solicitar permisos;
- solicitar areas;
- revisar el estado de sus solicitudes.

## Que puede hacer administracion

La administracion del sistema puede:

- aprobar o rechazar registros de usuarios;
- revisar solicitudes de permisos;
- revisar solicitudes de areas;
- asignar areas;
- administrar catalogos;
- consultar auditoria;
- dar seguimiento al gobierno documental.

## Que hace diferente a este sistema

No se trata solo de un listado de archivos. La diferencia real esta en que el sistema integra piezas que normalmente estan separadas:

- seguridad,
- versionado,
- flujo documental,
- busqueda,
- OCR,
- auditoria,
- despliegue y observabilidad.

Eso le da mayor madurez tecnica y tambien mayor utilidad operativa.

## Por que importa la busqueda

En organizaciones con muchos documentos, el problema no es solo almacenar, sino encontrar.

El sistema resuelve eso permitiendo buscar por:

- nombre,
- codigo,
- categoria,
- tipo,
- area,
- y contenido.

Cuando hay PDFs escaneados, el sistema puede aplicar OCR para que esos documentos tambien entren a la busqueda.

## Por que importa la auditoria

En sistemas con informacion sensible o regulada, no basta con permitir acciones. Tambien hay que dejar evidencia de ellas.

Por eso el sistema registra eventos importantes, por ejemplo:

- inicios de sesion;
- accesos denegados;
- busquedas;
- aprobaciones;
- cambios administrativos;
- operaciones sobre documentos.

## Que valor aporta el proyecto

El valor del proyecto puede resumirse en cuatro ideas:

1. organiza la informacion;
2. reduce ambiguedad;
3. mejora la trazabilidad;
4. hace mas eficiente la consulta y operacion documental.

## Como conviene explicarlo en una defensa

Una forma sencilla y fuerte de explicarlo seria:

"DMS SIG es una plataforma de gestion documental que centraliza documentos, controla permisos y areas, mantiene historial de versiones, soporta revision y aprobacion, integra busqueda avanzada con Elasticsearch y OCR para PDFs escaneados, y deja evidencia mediante auditoria."

Esa frase condensa bastante bien el valor del sistema sin entrar todavia en la parte mas tecnica.
